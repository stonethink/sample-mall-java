## 详细修改清单

### P0 问题修复（4个）

#### 1. ✅ 补充用户端展示设计（原问题1）

**修改位置：** 新增章节 6. 用户端展示设计

**新增内容：**
- 商品详情页促销标签展示（含倒计时、活动规则）
- 购物车页面优惠计算展示（含优惠明细、差额提醒）
- 订单确认页优惠明细展示

**关键设计：**
- 仅展示 PREHEATING 或 ACTIVE 状态的活动
- 按优先级排序，最多展示 2 个活动
- 预热状态显示"即将开始"，进行中的显示倒计时

---

#### 2. ✅ 修复活动冲突校验规则（原问题2）

**修改位置：** 5.2 活动冲突校验规则

**核心变更：**
```
原规则：只判断商品范围交集
新规则：冲突 = (商品范围有交集) AND (时间范围有重叠) AND (两个活动都启用)
```

**新增内容：**
- 时间重叠判定算法
- 商品范围交集判定算法
- 冲突处理策略（拒绝创建/自动调整/强制创建）
- 8个冲突场景示例（新增场景6-8覆盖时间维度）

---

#### 3. ✅ 明确订单金额计算逻辑（原问题3）

**修改位置：** 3.4 订单扩展、5.4 优惠计算规则

**核心变更：**
```
原逻辑：payAmount = originalAmount - promotionDiscount
新逻辑：payAmount = originalAmount - promotionDiscount - couponDiscount + freightAmount
```

**新增字段：**
- `freightAmount`: 运费金额（不参与满减计算）
- `couponDiscount`: 优惠券抵扣金额
- `ruleSnapshot`: 规则快照（JSON），记录下单时的活动规则

**明确规则：**
- 商品单价取下单时刻的快照价格
- 满减计算基数**不包含运费**
- 优惠叠加顺序：满减优先于优惠券
- 最小支付金额为 0

---

#### 4. ✅ 修复 Swagger 兼容性（原问题9）

**修改位置：** 2.3 技术选型、10.3 相关资源

**核心变更：**
```
原技术：Swagger 3.0.0 (Springfox)
新技术：SpringDoc OpenAPI 1.7.0
```

**影响：**
- Swagger UI 路径变更：`/swagger-ui.html` → `/swagger-ui/index.html`
- 解决 Spring Boot 2.7+ 兼容性问题

### P1 问题修复（5个）

#### 5. ✅ 增加活动优先级机制（原问题4）

**修改位置：** 3.2 促销活动实体

**新增字段：**
- `priority`: Integer，活动优先级，数值越大优先级越高，默认 0

**影响范围：**
- 活动列表支持按优先级排序
- 优惠计算时优先应用高优先级活动
- 用户端展示按优先级排序

---

#### 6. ✅ 支持每满减类型（原问题5）

**修改位置：** 3.3 满减规则实体、5.4 优惠计算规则

**新增字段：**
- `type`: Enum，规则类型（LADDER 阶梯 / PER_AMOUNT 每满）

**新增枚举：**
```java
public enum RuleType {
    LADDER,      // 阶梯满减：满100减10，满200减30（取最高档位）
    PER_AMOUNT   // 每满减：每满100减10，可叠加（200元减20）
}
```

**计算规则对比：**
| 类型 | 订单¥250 | 计算方式 | 优惠金额 |
|------|----------|----------|----------|
| LADDER | 满200减30 | 取最高档位 | ¥30 |
| PER_AMOUNT | 每满100减10 | 可叠加 | ¥20 |

---

#### 7. ✅ 明确接口分页和筛选参数（原问题7）

**修改位置：** 4.1 REST API 清单、4.2.3 新增接口

**新增接口：**
- `GET /api/promotions` 支持分页查询

**请求参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | Integer | 否 | 页码，默认1 |
| size | Integer | 否 | 每页条数，默认20，最大100 |
| status | String | 否 | 状态筛选 |
| keyword | String | 否 | 关键词搜索（活动名称） |
| sort | String | 否 | 排序字段，如：startTime,desc |

---

#### 8. ✅ 增加异常处理和降级策略（原问题10）

**修改位置：** 新增章节 5.5 异常处理与降级策略

**新增内容：**
- 4种异常场景及处理策略
- 降级机制设计（熔断器模式）
- 告警与监控方案

**降级策略：**
| 异常场景 | 处理策略 | 用户感知 |
|----------|----------|----------|
| 优惠计算服务异常 | 允许下单，不享受优惠 | 提示"优惠活动暂不可用" |
| 活动规则数据异常 | 跳过该活动，继续匹配其他 | 无感知 |

---

#### 9. ✅ 完善订单扩展字段设计（原问题11）

**修改位置：** 3.4 订单扩展

**新增字段：**
- `freightAmount`: 运费金额
- `couponDiscount`: 优惠券抵扣金额
- `ruleSnapshot`: 规则快照（JSON）

**新增章节：** 历史数据兼容方案

| 字段 | 历史订单（无促销） | 新订单 |
|------|-------------------|--------|
| `originalAmount` | 等于 `totalAmount` | 商品原始金额 |
| `freightAmount` | 0 | 实际运费 |
| `promotionDiscount` | 0 | 实际满减优惠 |
| `couponDiscount` | 0 | 实际优惠券抵扣 |
| `payAmount` | 等于 `totalAmount` | 计算后的支付金额 |
| `promotionId` | null | 命中的活动ID |
| `ruleSnapshot` | null | 下单时的规则JSON快照 |

### P2 问题修复（3个）

#### 10. ✅ 增加活动预热机制（原问题6）

- `preheatTime`: LocalDateTime，预热开始时间（可选）
